## 二、问题根因分析

### BUG-1: HP 恢复远大于事件伤害，角色几乎不可能被事件杀死 [P1]

**根因**: `regenHp()` 的恢复量设计导致事件伤害被完全吸收。

分析 `regenHp()` 方法:
```
regen = initialStrRegen = 1 + floor(initStr / 3)
```
- 人类平均初始 str ≈ 7 → regen = 3-4/年
- 矮人平均初始 str ≈ 12 → regen = 5/年
- 精灵平均初始 str ≈ 3 → regen = 2/年

**事件伤害统计**（235 个 modify_hp 效果）:
- 最大单次伤害: -55（瘟疫失败分支）
- 70% 的伤害在 -1 到 -10 之间
- 年均事件触发数约 1-1.2 个（基于平均事件数/寿命）
- **年均事件伤害 ≈ -3 到 -5 HP**
- 但 **年均恢复 = regen（3-5）- ageDecay（0 直到 68% 寿命）**

**结论**: 在衰老期之前（lifeRatio < 0.68），角色每年净恢复 3-5 HP。事件伤害（年均 -3 到 -5）**与恢复基本持平甚至更小**，意味着事件几乎不可能把角色 HP 打到 0。

即使触发最大伤害事件（-55 HP），角色有 10-20 年恢复时间，足以从 0 恢复到满 HP。

### BUG-2: 衰老阈值太晚太慢，无法在寿命范围内可靠致死 [P1]

**根因**: ageDecay 的阈值设计不合理。

当前衰减阶梯:
```
lifeRatio >= 0.68 → ageDecay = 1
lifeRatio >= 0.78 → ageDecay = 2
lifeRatio >= 0.85 → ageDecay = 3
lifeRatio >= 0.92 → ageDecay = 4
lifeRatio >= 1.00 → ageDecay = 7
lifeRatio >= 1.10 → ageDecay = 10
```

**问题**:
1. **衰减开始太晚**: lifeRatio=0.68 意味着人类要到 54 岁才开始衰减，矮人要到 108 岁
2. **衰减太温和**: 即使到 lifeRatio=0.92（人类 78 岁），ageDecay=4 仍然小于 regen=4 → **净恢复为 0**
3. **只有超过寿命上限后才加速**: lifeRatio>1.0 时 ageDecay=7，对 regen=3 的精灵终于产生净亏损（-4/年），但此时 HP 软上限还有 30-40

**计算**: 对于 regen=3 的角色，lifeRatio=1.0 时 softCap≈30，HP≈50-60
- 每年净亏损: 3 - 7 = -4
- 需要约 12-15 年才死亡 → 最终寿命 ≈ 112-115% 寿命上限
- 这完美解释了矮人（114-115%）、半龙人（112%）的数据

### BUG-3: HP 软上限在后期太高，衰减效果被稀释 [P2]

softCap 在 lifeRatio=1.0 时为 `initHp × 0.7`:
- 人类: 35 × 0.7 ≈ 25
- 精灵: 37 × 0.7 ≈ 26
- 矮人: 67 × 0.7 ≈ 47

最低 softCap 为 20（硬下限）。当角色开始衰老时，softCap 只比 HP 低一点点，导致衰减的大部分效果被"在 softCap 之上"的区域浪费。

### BUG-4: 评分公式中寿命×0.3 对长寿种族的指数级加成 [P1]

评分公式:
```
score = attrPeakSum × 1.2 + lifespan × 0.3 + itemCount × 3 + routeBonus
```

**表面上看**寿命分占比只有 5-13%，但这忽略了**长寿种族有更多年积累属性**。

实际数据:
- 精灵平均属性峰值: 822（人类 283，**2.9 倍**）
- 精灵平均寿命: 386 年（人类 89.7 年，**4.3 倍**）
- 精灵属性分: 987（人类 340，**2.9 倍**）
- 精灵寿命分: 116（人类 27，**4.3 倍**）
- **综合: 精灵 1103 vs 人类 367 = 3.0 倍**

**根因**: 不是寿命分单独的问题，而是**超长寿命让属性积累碾压**。380 年 vs 80 年意味着 4.75 倍的事件触发机会，属性自然远高于短寿种族。

### BUG-5: 极端早夭的随机方差 [P2]

**根因**: 事件 `child_drowning`（河中溺水，age 5-12）等童年伤害事件对低 str 种族致命。

- 精灵初始 str=2（默认5-3），初始 HP=25+2×3=31
- `child_drowning` 失败分支 HP-45 → HP 从 31 降到 0（-14，触发致命打击再 -10）
- 精灵有 8 局（20%）寿命 < 50 岁，全部是童年/少年阶段被事件杀死
- 海精灵初始 str=2，同样脆弱

**这不是"平衡性"问题，而是缺少低龄阶段的 HP 安全机制**。

## 三、调整建议

### 优先级 P0（必须立即修复）

#### 3.1 重写 regenHp 衰老曲线

**目标**: 角色在 lifeRatio=0.85-0.95 之间自然死亡，不超出寿命上限

**建议实现**（替换整个 ageDecay 阶梯为连续函数）:

```typescript
// 基于 lifeRatio 的连续衰减函数
// 目标：对 regen=3 角色，在 lifeRatio≈0.88 时 net=0；regen=5 角色在 lifeRatio≈0.92 时 net=0
let ageDecay = 0
if (lifeRatio >= 0.55) {
  // 从 lifeRatio=0.55 开始轻微衰减，加速到 lifeRatio=1.0
  const decayProgress = Math.min((lifeRatio - 0.55) / 0.45, 1.0)
  // 二次曲线：前期慢，后期快
  ageDecay = regen * (0.2 + 1.2 * decayProgress * decayProgress)
  ageDecay = Math.ceil(ageDecay)
}

// 超出寿命上限：强力衰减
if (lifeRatio > 1.0) {
  const overage = lifeRatio - 1.0
  ageDecay += Math.floor(regen * 2 * overage + 5)
}
```

**预期效果**:
- regen=3 角色: lifeRatio≈0.60 时 net≈0, 死于 lifeRatio≈0.85
- regen=4 角色: lifeRatio≈0.65 时 net≈0, 死于 lifeRatio≈0.88
- regen=5 角色: lifeRatio≈0.70 时 net≈0, 死于 lifeRatio≈0.92
- 超出 lifeRatio=1.0 后额外 -5+ HP/年，确保不死

#### 3.2 降低初始 HP 公式中的 str 系数

**当前**: `initHp = 25 + str × 3`
**建议**: `initHp = 20 + str × 2`

**理由**: 降低初始 HP 缓冲区，让童年危险事件（如 -45 HP 的溺水）需要更多运气才能存活，但 regen=2 的精灵仍有约 24 HP，不至于 100% 死亡。

#### 3.3 降低软上限下限

**当前**: `Math.max(softCap, 20)` — 最低 20 HP
**建议**: `Math.max(softCap, 10)` — 最低 10 HP

**理由**: 软上限下限 20 意味着即使在 lifeRatio=1.0 时，精灵仍有 softCap≈26。降低到 10 使后期衰减更致命。

### 优先级 P1（尽快修复）

#### 3.4 重做评分公式，消除长寿种族的指数级优势

**当前公式**:
```
score = attrPeakSum × 1.2 + lifespan × 0.3 + itemCount × 3 + routeBonus
```

**建议公式**（引入寿命归一化和属性效率）:
```typescript
const visibleAttrs = world.attributes.filter(a => !a.hidden && a.group !== 'hidden')
const attrCount = visibleAttrs.length

// 属性效率分：每属性平均峰值（消除"活越久属性越高"的线性优势）
const avgPeakPerAttr = totalAttributePeakSum / attrCount

// 属性效率分使用平方根曲线，压缩高数值
const attrScore = Math.sqrt(avgPeakPerAttr) * 18  // 18 是调节系数

// 寿命分：基于 lifespan/lifespanRange 上限的比例，使用对数压缩
const lifeRatio = Math.min(lifespan / effectiveMaxAge, 1.5)
const lifespanScore = Math.log2(lifeRatio + 1) * 50  // 对数压缩

const score = attrScore + lifespanScore + itemCount * 5 + routeBonus
```

**预期效果**:
- 精灵 avgPeakPerAttr≈117, attrScore≈√117×18≈195
- 人类 avgPeakPerAttr≈40, attrScore≈√40×18≈114
- 精灵寿命分: log2(1.05+1)×50≈51；人类: log2(1.04+1)×50≈51
- **精灵总分 ≈ 246 vs 人类 ≈ 165 — 约 1.5 倍差距（可接受）**

#### 3.5 增加"高风险事件"权重，提供更多提前死亡机会

当前事件系统中，大多数战斗/冒险事件的伤害在 -10 到 -35 之间（且需要选失败分支才触发）。建议:

1. **增加"暴击死亡"机制**: 在 `modify_hp` 处理中，当单次伤害 > 当前 HP 的 80% 时，有 30% 概率直接致死（不取决于 HP 是否归零）
2. **增加中年期随机致命事件**: 在 middle-age.json 中增加 3-5 个"意外死亡"事件（如坍塌、瘟疫、野兽袭击），weight=2-3，效果为 modify_hp=-30~-40

**理由**: 消除精灵/海精灵因童年事件随机暴毙的体验问题，同时保留一定风险感。
