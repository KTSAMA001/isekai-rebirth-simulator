> 调研日期：2026-03-30
> 目标：为「剑与魔法」D&D风格异世界模拟器设定合理的死亡风险分布

## 一、调研数据汇总

|------|------|------|
| 人类 | ~100年 | 基准 |
| 兽人(Orc) | ~75年 | 最短 |
| 龙裔(Dragonborn) | ~80年 | |

| 矮人(Dwarf) | ~350年 | |

**关键结论**：人类基准寿命约100岁，但这是「自然上限」，实际平均寿命受死亡风险影响远低于此。

### 1.2 D&D 冒险者死亡率

根据社区调查（Reddit r/dndnext 2020年调查）：

- **约75%的PC死亡来自战斗**（包括正常战斗、Boss战、遭遇战）
- 死亡原因分布：
  - 正常战斗：~40%
  - Boss战/关键战斗：~25%
  - 陷阱/环境危险：~15%
  - 疾病/中毒：~10%
  - 其他（意外、诅咒等）：~10%
- **D&D各版本的致死率趋势**：1E > 2E > 3E > 4E > 5E（5e是最温和的版本）
- **死亡豁免存活率**：单次倒地后有约 **59.5%** 的概率自行稳定（3次成功vs3次失败）

**低等级(1-5级)风险特征**：
- 1级角色HP仅 8-12（取决于职业和体质），CR 1/4的地精一刀可造成 5-8 伤害
- 一个CR 1的怪物对1级队伍构成「困难」遭遇
- 1-3级是死亡率最高的阶段（被称为「肉 grinder levels」）
- 估计约 **30-40%** 的1级角色会在前3级内死亡（根据社区反馈）

- 饥荒/营养不良：10%
- 衰老：5-10%

### 1.4 BitLife 等人生模拟器的死亡设计

**BitLife 死亡机制**：
- 老年死亡：60岁+开始有概率因「年老并发症」死亡，概率随年龄递增
- 120岁+可获得「Geriatric」绶带（说明上限很高但极罕见）
- 疾病死亡：任何年龄段，与健康值强关联
- 意外死亡：随机但低概率（雷击、恐怖袭击等）
- 行为致死：嗑药、犯罪、冒险选择等高概率致死
- 家庭成员死亡主要发生在65岁+
- **平均寿命**：玩家报告约 32-70岁不等（取决于玩法和健康值管理）

**设计哲学**：
- 早期（0-30岁）：死亡主要来自疾病和极端事件，概率低
- 中期（30-60岁）：死亡风险缓慢上升，主要由玩家选择驱动
- 后期（60岁+）：死亡概率急剧上升，自然衰老成为主因
- 核心原则：**让大部分玩家能活到中年体验核心内容，但不保证能活到老年**

## 二、综合分析与建议

### 2.1 目标寿命分布（剑与魔法世界）

基于以上调研，考虑到：
- 这是一个**D&D风格的冒险世界**，危险度高于普通中世纪
- 但有**魔法治疗**（牧师、药水），可抵消部分疾病/伤害致死
- 作为**游戏**，需要让大部分玩家体验到青年期以后的核心内容
- 考虑到游戏设定的 **maxAge=100**

#### 建议的存活率分布

```
年龄区间    存活到该阶段%    该阶段死亡%    主要死因
───────────────────────────────────────────────────────────
出生        100%            ~3%            难产/先天不足
童年(3-10)  97%             ~5%            疾病/意外/怪物袭击
少年(11-17) 92%             ~4%            霸凌/冒险/事故
青年(18-27) 88%             ~8%            战斗/冒险/试炼
壮年(28-50) 80%             ~15%           战争/魔物/政治阴谋
中年(51-69) 65%             ~25%           疾病/旧伤/衰弱
老年(70+)   40%             ~35%           自然衰老/疾病
───────────────────────────────────────────────────────────
活过70岁    ~26%  ← 约1/4的NPC能活到"长寿"
活过85岁    ~8%   ← 极少数
达到100岁   ~1%   ← 罕见的传奇长寿
```

#### 目标平均寿命

| 指标 | 建议值 | 理由 |
|------|--------|------|
| **普通NPC平均寿命** | ~55岁 | 有魔法但危险世界 |
| **冒险者平均寿命** | ~40岁 | 高风险职业 |
| **冒险者中位数寿命** | ~35岁 | 很多人在新手期阵亡，拉低了均值 |
| **安全职业（学者/商人）** | ~65岁 | 接近中世纪上层阶级水平 |

### 2.3 对当前模拟器的具体调整建议

#### 建议 1：建立分年龄段的「基础死亡风险」机制

当前系统主要靠 HP 归零来判定死亡，但缺少一个**每年递增的基础死亡概率**。

```
建议在每轮事件结算后，额外进行一次「年度存活检定」：

年龄段    基础死亡概率/轮    说明
──────────────────────────────────
3-10      0.3%              略高于现代，远低于中世纪
11-17     0.3%              少年期相对安全
18-27     0.5%              冒险开始，风险上升
28-50     1.0%              冒险黄金期，风险显著
51-69     2.0%              身体机能下降
70-79     5.0%              老年风险急升
80-89     10%               高龄
90-99     20%               极高龄
100       50%               传奇年龄
```

**修正因子**：
- 体质(Con)加成：每点 Con +1 以上 → 死亡概率 ×0.9
- 魔法治疗：有治疗药水/牧师 → 死亡概率 ×0.7
- 冒险者职业：死亡概率 ×1.3（高风险生活方式）
- 已婚/有家庭：死亡概率 ×0.9（求生欲加成）
- 旧伤/慢性病flag：死亡概率 ×1.5

#### 建议 2：事件中的死亡风险分级

对照当前事件的 `riskCheck` 和 `modify_hp` 系统，建议统一死亡风险等级：

```
风险等级    HP损失占比    死亡概率    适用场景
─────────────────────────────────────────────
微小(minor)    <10%       ~1%       日常事件、小冲突
低(low)       10-25%      ~3%       一般战斗、训练事故
中(medium)    25-50%      ~8%       认真战斗、危险任务
高(high)      50-75%     ~15%       Boss战、致命陷阱
致命(lethal)   >75%      ~25%       屠龙、禁忌魔法、决斗
```

#### 建议 3：具体事件权重调整

| 年龄段文件 | 当前事件数 | 建议增加的死亡/濒死事件 | 建议调整 |
|-----------|-----------|----------------------|---------|
| childhood.json | ~30个 | +2-3个（疾病、魔力暴走失控） | 死亡事件weight=1-2 |
| teenager.json | ~40个 | +3-4个（试炼事故、霸凌升级） | 死亡事件weight=1-3 |
| youth.json | ~50个 | +5-6个（冒险战斗、中毒） | 死亡事件weight=2-4 |
| adult.json | ~70个 | +4-5个（战争、暗杀） | 死亡事件weight=2-5 |
| middle-age.json | ~30个 | +3-4个（旧伤、衰弱） | 死亡事件weight=3-6 |
| elder.json | ~20个 | +2-3个（疾病、安详离世） | 死亡事件weight=5-10 |

#### 建议 4：引入「寿命修正」种族特性

在 `rules.json` 或 `manifest.json` 中增加种族寿命修正：

```
种族        寿命修正    maxAge实际值    说明
─────────────────────────────────────────────
人类         ×1.0       100           基准
精灵         ×7.5       750           极长寿，但需要特殊处理
矮人         ×3.5       350           长寿种族
半精灵       ×1.5       150           中等长寿
兽人         ×0.75      75            短寿但强悍
龙裔         ×0.8       80            略短于人类
```

**注意**：对于寿命远超人类的种族（精灵、矮人），需要重新设计各年龄段的触发时间，否则会出现「精灵玩了200轮还在青年期」的问题。建议简化处理：**所有种族共用人类的时间轴，长寿种族通过降低死亡概率来体现寿命优势**。

#### 建议 5：游戏体验平衡原则

基于 BitLife 等人生模拟器的设计经验：

1. **「黄金法则」— 至少70%的玩家应能活到成年(18岁)**
   - 童年/少年的死亡事件应该是罕见的
   - 让玩家体验到核心玩法（职业选择、冒险系统）

2. **「有意义的选择」— 成年后的死亡应主要来自玩家选择**
   - 选择冒险路径 → 高风险高回报
   - 选择安全路径（学者、商人） → 低风险但人生可能平淡
   - 让玩家觉得「是我自己选的」而非「随机死了」

3. **「善终率 25-30%」— 约1/4的NPC应能安享晚年**
   - 给追求「完美人生」的玩家足够的成功感
   - 老年期事件以温情和回顾为主，不是连续死亡检定

4. **「突然死亡的震撼值」— 偶尔的意外死亡增加游戏张力**
   - 约每5-8次游戏出现一次「意外早逝」
   - 但应给出明确的叙事原因（而非「你不幸死了」）
   - 参考：elder_final_counting 事件做得很好，可扩展到其他年龄段

5. **「HP 作为缓冲」— 不要让单次事件直接致死**
   - 当前 HP 系统已有一定缓冲
   - 建议：HP 归零时触发「濒死事件」（如 near_death flag），给玩家一次选择机会
   - 而非直接判定死亡

## 三、数值公式建议

### 3.1 每轮死亡概率计算

```
P(死亡) = 基础概率(年龄) × 种族修正 × 体质修正 × 生活方式修正 × 事件修正

其中：
  基础概率(年龄) = 参见 2.3 建议的表格
  种族修正 = 1.0(人类) / 0.6(矮人) / 0.4(精灵) / 1.3(兽人)
  体质修正 = max(0.5, 1.0 - (Con-10)×0.03)   ← Con每高1点降低3%
  生活方式修正 = 0.7(和平) / 1.0(普通) / 1.5(冒险者) / 2.0(前线战士)
  事件修正 = 该轮事件的风险等级系数
```

### 3.2 HP 与死亡的关系

```
当前HP > 50%   → 正常状态，不额外增加死亡概率
当前HP 25-50%  → 基础死亡概率 ×1.5
当前HP 1-25%   → 基础死亡概率 ×3.0
当前HP = 0     → 触发濒死事件（非直接死亡）
```

## 四、优先级建议

| 优先级 | 任务 | 理由 |
|--------|------|------|
| P0 | 在 rules.json 中定义年龄段基础死亡率 | 最核心的框架 |
| P0 | 为 HP=0 添加濒死事件链 | 防止突然死亡影响体验 |
| P1 | 为各年龄段添加 2-3 个死亡/濒死事件 | 丰富死亡叙事 |
| P1 | 添加种族寿命修正 | 区分种族差异 |
| P2 | 添加 Con 属性对死亡率的影响 | 让属性有意义 |
| P2 | 事件风险等级标签标准化 | 方便后续平衡调整 |
| P3 | 统计系统：记录平均死亡年龄和死因 | 用于持续优化 |
